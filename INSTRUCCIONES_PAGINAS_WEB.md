# Instrucciones para modificar páginas web

1. Identificar exactamente la página, ruta y cambio solicitado.
2. Leer primero el archivo actual y los CSS, JavaScript o archivos base que cargue.
3. Hacer el cambio mínimo necesario; no modificar otras páginas ni funciones.
4. Si la página usa un cargador y un archivo base, conservar esa estructura y editar el archivo correcto.
5. Mantener diseño adaptable a celular, accesibilidad, enlaces, formularios, estadísticas y SEO existentes.
6. Para contenido destinado a Google, usar HTML semántico y texto real visible o desplegable; no ocultarlo artificialmente.
7. Si cambia un CSS o JavaScript enlazado, actualizar su parámetro de versión para evitar caché antigua.
8. No borrar productos, datos, imágenes, históricos ni archivos sin una orden expresa.
9. No dejar archivos, enlaces, ramas o incidencias temporales después de verificar.
10. Revisar sintaxis, carga de recursos, texto visible y funcionamiento antes de terminar.
11. Trabajar en la rama indicada; si no se indica, usar la rama predeterminada del repositorio.
12. Al finalizar, informar archivos modificados, resultado, enlace público y conteo:

Original = líneas anteriores
Entregado = líneas finales
Variación = diferencia
